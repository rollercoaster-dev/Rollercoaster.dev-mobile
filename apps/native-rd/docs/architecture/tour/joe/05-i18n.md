## Scope

Runtime i18n only — how the app loads, selects, and renders localized strings. The LLM-driven translation pipeline at `scripts/i18n/` is reviewed separately in slice 8 (build pipeline & tooling), because tooling has its own lens (LLM gateway, glossary management, promptfoo, sync orchestration) that's distinct from runtime concerns (provider, language detection, key coverage, namespace organization).

**In scope:**

- `src/i18n/index.ts` — i18next initialization and config
- `src/i18n/language.ts` — language/locale switching logic
- `src/i18n/pseudoTransform.ts` — pseudo-locale transformation for QA
- `src/i18n/labels.ts` — hardcoded fallback labels (legacy — flag for migration)
- `src/i18n/i18next.d.ts` — TypeScript namespace augmentation for resource keys
- `src/i18n/resources/en/**` — English source (20 namespaces, ~84 KB)
- `src/i18n/resources/de/**` — German translations
- `src/i18n/resources/pseudo/**` — pseudo-locale (generated by tooling, reviewed here for runtime correctness)
- `src/i18n/resources/_register/` — locale registration metadata

**Deferred:**

- Translation sync pipeline (`scripts/i18n/**`: sync.cli, lintSource.cli, glossary/intent loaders, LLM gateway, promptfoo) — slice 8
- Pseudo-locale generator script — slice 8
- Per-screen string audits — surface during domain slices (6, 7, 9), file findings against slice 5 if they're systemic
