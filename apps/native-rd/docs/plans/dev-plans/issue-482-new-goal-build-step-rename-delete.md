# Development Plan: Issue #482

## Issue Summary

**Title**: [Storybook] New Goal build step — inline rename + delete on step rows
**Type**: feature (Storybook-only, follow-up to #464)
**Complexity**: SMALL–MEDIUM
**Estimated Lines**: ~340–450 lines across 4 files (component + styles + stories + tests) — see "Decisions" D2 for why this exceeds the issue's own ~150–250 estimate.

## Intent Verification

- [x] Tapping a build-step row's title (in `NewGoalWizard`'s `build` step) enters inline-edit mode — the title becomes a focused `TextInput` seeded with the current title — exactly mirroring `EditGoalStepRow`'s tap-to-edit idiom (same a11y label shape `Edit step: <title>`, same hint `Tap to edit step title`).
- [x] Submitting (return key) or blurring the inline edit commits the new title via a caller-owned callback; the row returns to display mode showing the updated title.
- [x] Each build-step row exposes a `×` delete affordance (same glyph/style idiom as `EditGoalStepRow`'s `stepDelete`/`stepDeleteGlyph`), hidden while that row is mid-rename.
- [x] Pressing `×` opens the shared `ConfirmDeleteModal` (the same component `EditGoalView` uses, per the #460 "all deletions must be confirmed" directive); the row is removed from the list **only on Confirm**, never on the raw `×` press.
- [x] The build list stays flat: no row exposes a nesting/sub-step affordance, and `BuildStep` gains no `parentStepId`-shaped field.
- [x] `NewGoalWizard.stories.tsx`'s `BuildStep` (and the end-to-end `InteractiveFlow`) stories demonstrate rename and delete-confirm working end to end against local story state; a frozen story shows the delete-confirm modal open (mirrors `Step2PickerOpen`'s "no interaction needed to see it" pattern).
- [x] No hardcoded hex in any new style — only `theme.colors.*`/`theme.space.*` tokens via `react-native-unistyles` (this app's actual equivalent of the issue's stale `--ob-*` reference; see Decisions D5).

## Objective

Add inline rename and a confirmed delete affordance to each row on `NewGoalWizard`'s build-list step (`currentStep === "build"`), reusing `EditGoalStepRow`'s rename idiom and the shared `ConfirmDeleteModal` (per the #460 precedent), without introducing nesting or persistence. Storybook/component scope only — `[Integrate]` (#444) wires real Evolu mutations later.
